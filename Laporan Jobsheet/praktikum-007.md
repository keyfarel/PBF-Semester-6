## Pertanyaan Evaluasi

**1. Apa fungsi API Routes pada Next.js?**  
API Routes pada Next.js berfungsi untuk membuat endpoint API di dalam aplikasi Next.js (sebagai *backend*) tanpa perlu membuat server terpisah. File yang berada di dalam folder `pages/api` akan otomatis dipetakan menjadi rute backend (contohnya `/api/produk`). Hal ini memungkinkan kita untuk menjalankan operasi dan logika di sisi server (*server-side*) dengan aman, seperti berinteraksi dengan database.

**2. Mengapa .env.local tidak boleh di-push ke repository?**  
File `.env.local` berisi konfigurasi *environment variables* yang menyimpan data rahasia dan sensitif (*credentials*), seperti Access Token, API Key untuk Firebase, *password* atau *URL database*. Jika di-push ke repository (terutama repositori publik seperti GitHub), kredensial tersebut akan terekspos ke semua orang dan berpotensi dicuri lalu disalahgunakan.

**3. Apa perbedaan data statis dan data dinamis?**  
- **Data Statis:** Data yang sifatnya tetap dan tidak dapat diperbarui setelah aplikasi dijalankan. Data ini biasanya tertanam secara langsung (*hardcoded*) di dalam *source code*.  
- **Data Dinamis:** Data yang secara aktif bisa ditambah, diubah, maupun dihapus kapan saja mengikuti interaksi pengguna. Data ini tidak di-*hardcode* melainkan disimpan pada database eksternal (seperti Firestore) dan selalu di-*fetch* (diminta) ulang secara *real-time* ke server oleh aplikasi ketika akan diakses.

**4. Mengapa Next.js disebut framework fullstack?**  
Next.js disebut sebagai *framework fullstack* karena ia tidak hanya bekerja di sisi *frontend* (menggunakan React.js untuk merancang UI), tetapi ia juga menyediakan fungsionalitas untuk menyusun sisi *backend* (Server-side/API) melalui fitur unggulannya yaitu **API Routes**. Oleh karena itu, *developer* dapat membangun *user interface* beserta pengolahan data *server* secara lengkap cuma dalam satu proyek (*codebase*) Next.js yang sama.
